# ERIC888 核心板 B（面板板/Panel）全功能开发清单

> **目标**：在 STM32F429 上实现高性能 GUI 交互、多规约物联网通讯及海量数据录波。
> **遵循准则**：.antigravity_rules.md

---

## 一、 基础驱动层 (Hardware Abstraction Layer)

### 1. LTDC RGB 屏驱动
- [ ] **配置 LTDC 外设**：根据 800x480 分辨率配置 HSYNC/VSYNC/DE 时序。
- [ ] **SDRAM 帧缓冲**：在 SDRAM (0xC0000000) 映射双层显存，实现无撕裂显示。
- [ ] **DMA2D 加速**：实现矩形填充、图层混合及格式转换的硬件加速。
- [ ] **背光控制**：PA11 (TIM1_CH4) 实现 0-100% 亮度调节。

### 2. STMPE811 触摸驱动
- [ ] **I2C1 通讯**：实现对触摸芯片寄存器的读写。
- [ ] **中断响应**：PH7 (TP_INT) 触发坐标读取，避免主循环轮询。

### 3. W5500 以太网驱动
- [ ] **SPI2 适配**：实现 W5500 的底层读写接口。
- [ ] **Socket 管理**：支持 Modbus TCP 协议，响应上位机查询。

### 4. SDIO + FatFs 存储驱动
- [ ] **4-bit SDIO 模式**：配置 SDIO 接口，主攻高速读写。
- [ ] **FatFs 挂载**：实现文件系统，用于存储历史故障波形（.dat格式）。

### 5. EC20 4G 模块通讯
- [ ] **UART7 AT 解析器**：实现 AT 指令发送与 URC 消息异步处理。
- [ ] **电源控制**：PF8 (4G_POW) 软开关逻辑。

---

## 二、 业务逻辑层 (Application Layer)

### 1. 板间通信桥梁 (SPI4 Master)
- [ ] **SPI4 DMA 定时读取**：作为 Master 轮询 A 板（Slave），获取 25.6kHz 的采样数据流。
- [ ] **数据帧解析**：验证包头、CRC16，解出 8 通道 ADC 原生数据。
- [ ] **故障同步**：识别 A 板发出的 TRIP 信号，立即切换 UI 状态并启动本地录波。

### 2. LVGL 交互界面
- [ ] **实时示波器**：绘制 8 通道电流电压实时曲线（需利用硬件加速保证流畅度）。
- [ ] **参数配置页**：通过 UI 修改 OC/OV 阈值，并通过 SPI4 同步给 A 板。
- [ ] **故障查询页**：读取 SD 卡文件并进行波形复现。

### 3. 规约转换中枢
- [ ] **Modbus RTU/TCP**：支持通过 UART4/2 及以太网读取系统状态。
- [ ] **GPS 定位同步**：UART1 读取 NMEA 数据，为故障记录提供精准地理位置标签。

---

## 三、 健壮性与验证 (Verification)

### 1. 监控系统
- [ ] **Watchdog (IWDG)**：确保在 UI 挂起或通讯死锁时系统重启。
- [ ] **CPU 负载审计**：监控 LVGL 渲染耗时与通讯任务占空比。

### 2. 自动化测试 (ACP 运行目标)
- [ ] **make build**：确保全工程无警告编译。
- [ ] **协议压力测试**：模拟 A 板持续发送 115.2kbps 负载，检查 CRC 错误率。
