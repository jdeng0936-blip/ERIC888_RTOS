# ERIC888 工业级高速控制器 - 软件工程架构文档

## 1. 系统概述
本项目采用 **双 STM32F429 核心板** 架构，旨在实现微秒级硬实时保护与丰富的人机交互、物联网通信相分离的工业级设计（“一文一武”架构）。

* **核心板 A（计算板/Calc）**：主攻极速采样、保护逻辑计算与瞬时跳闸执行。
* **核心板 B（面板板/Panel）**：主攻屏幕显示、外部网络规约通信与数据存储。
* **跨板通信桥梁**：SPI4（通过 SIT3490 进行 RS-422 差分隔离传输）。

## 2. 工作区目录规划 (Monorepo)
为避免底层驱动冲突，双核代码在物理层面分离，在协议层面共享：

\`\`\`text
ERIC888_Workspace/
 │
 ├── 1_ERIC888_Calc/         # 【当前阶段重点】核心板A工程 (裸机实时/FreeRTOS)
 │   ├── ERIC888_Calc.ioc    # 计算板 CubeMX 配置文件
 │   ├── Makefile            # 指定 gcc 14.2 编译器的构建脚本
 │   └── Core/               # 包含 main.c, AD7606极速读取, 继电器控制代码
 │
 ├── 2_ERIC888_Panel/        # 【后续阶段】核心板B工程 (GUI/网络协议栈)
 │   ├── ERIC888_Panel.ioc   # 面板板 CubeMX 配置文件
 │   ├── Makefile            # 面板板专属构建脚本
 │   └── Core/               # 包含 LVGL, LwIP, W5500, SD卡读写代码
 │
 └── 3_Common_Protocol/      # 【双核共享桥梁】
     └── eric888_spi_protocol.h  # SPI 数据包格式、状态机枚举全局定义
\`\`\`

## 3. 硬件资源与任务分配矩阵

### 3.1 核心板 A：计算板 (硬实时层)
* **核心使命**：死磕速度，微秒级响应，不受任何外设阻塞。
* **关键外设绑定**：
  * **FMC 总线 (`NE4` 映射 `0xD0000000`)**：极速并行读取 AD7606 (8路电压/电流)。
  * **GPIO 触发**：读取状态 (`POSITION_ON/OFF`)，执行跳闸 (`K1-ON/OFF` JDQ输出)。
  * **SPI4 (主/从待定)**：将打包好的波形和故障信号发送给面板板。

### 3.2 核心板 B：面板板 (综合管理层)
* **核心使命**：交互流畅，通信不掉线，海量数据存储。
* **关键外设绑定**：
  * **LCD 接口**：RGB 并行接口驱动大屏。
  * **SPI2 (`PB12~15`)**：独占连接 W5500 以太网芯片 (IEC61850 / Modbus TCP)。
  * **UART / SDIO**：驱动 4G 模块、语音播报芯片及 SD 卡故障录波。
  * **SPI4 (从/主待定)**：接收计算板发来的实时波形流。

## 4. 跨板通信协议规划 (SPI4 + DMA)
两板之间通过高速 SPI4 交换数据。数据包 (Frame) 基础结构设计如下：
* `[0-1]` 包头：`0xAA 0x55`
* `[2]`   指令类型：`0x01` (常规波形) / `0x02` (跳闸告警) / `0x03` (握手心跳)
* `[3]`   数据长度
* `[4-N]` 载荷 (Payload)：如 8 通道 int16_t ADC 数据
* `[N+1]` CRC16 校验码

## 5. 常用开发与调试指令 (Mac 环境)

### 编译指令 (计算板目录)
\`\`\`bash
make clean
make -j4
\`\`\`
*注：已在 Makefile 中硬编码绑定 `arm-none-eabi-gcc 14.2.1` 绝对路径，规避系统环境污染。*

### J-Link 一键烧录脚本
在编译生成 `.bin` 后，执行以下脚本即可刷入核心板 A：
\`\`\`bash
cat << EOF > flash.jlink
h
loadbin build/ERIC888_RTOS.bin, 0x08000000
r
g
q
EOF
JLinkExe -device STM32F429IG -if SWD -speed 4000 -autoconnect 1 -CommanderScript flash.jlink
\`\`\`
