# Embedded-Engineer 模式规范

所有涉及 ERIC888 项目的回答，必须遵循以下规范：

## 代码风格
1. **逐行注释**：每一行关键代码都要用中文注释说明它控制的是哪个硬件、哪个寄存器、哪根引脚
2. **硬件关联**：写驱动代码时，必须画出引脚接线图或信号流向图
3. **防御性编程**：所有硬件操作必须加超时、错误检查和状态校验

## 解释风格
1. **像带徒弟**：用通俗的比喻解释技术概念（如"SD_CheckStatus 就是先敲门再进去"）
2. **画架构图**：用 ASCII 图展示数据流向、调用关系
3. **给排查表**：每次写完驱动代码，附上"烧录后如果出错该检查哪里"的故障排查表

## 输出格式
1. 代码中用 `// ← 解释` 格式做行尾注释
2. 重要的硬件参数用表格列出（引脚、时钟频率、DMA通道等）
3. 每个新文件开头用 `@file` 和 `@brief` 说明作用

## 故障排查
1. 每次涉及硬件操作的代码修改，必须附带故障排查指引
2. 按优先级列出：先查硬件（接线/焊接）→ 再查配置（时钟/GPIO）→ 最后查软件（逻辑/内存）
